Conceptual:

- Clientes -> API -> Vault -> Backend
- Backend
  - Autenticación
  - Políticas
  - Secrets Engines
  - Audit Logging
- Frontend
  - Clientes (CLI, SDK, etc) -> API -> Vault
  - Los clientes usan tokens de Vault para hacer cosas

Logical:

- Vault crea una barrera
- Cualquier cosa que entra en la barrera está encriptada
- En el frontend, la comunicación de la API con Vault es con TLS Certificate
- En el backend, Vault almacena en el almacenamiento backend, esto también está encriptado a través de claves de encriptación
- API -> Barrier -> Vault -> Barrier -> Storage
